- v6.11.53 — UX: surface stream errors instead of silently dropping them. The chat handler had no `error`/`agentic_error` branch, so a server-side failure left the user staring at a spinner with no feedback. - v6.11.52 — UX: render the agentic/coding loop in the chat UI. The interface had no handler for `agentic_*` stream events, so coding requests ("check my folder and fix the bug") showed a spinner then "(empty response)" — the agent worked invisibly and its answer was dropped. Now a live, collapsible work-log shows each tool as a friendly step (🗂️ scanning… → 📖 reading… → ✏️ editing… → ✅ verified → 🧐 double-checking), the final answer actually renders, and the header collapses to "Adam worked through N steps". - v6.11.51 — Editorial calibration + manipulation flag. (1) `AMNI_IDEOLOGY_SKEW` (default 0.20): a DISCLOSED, tunable EPISTEMIC lean — the web synthesis + manipulation flag apply ~20% more scrutiny to progressive moral-superiority framing and treat the conservative reading as the tiebreaker on contested VALUES questions, to counter the documented progressive skew of indexed content. STRICTLY BOUNDED: facts/science/math/history/quotes stay objective; honest evidence-based arguments are NEVER flagged (validated: honest progressive = 20, not flagged, even with skew on); and it is EPISTEMIC ONLY — it never touches who Adam protects or refuses to harm (Law 0 + the physical-harm gate stay viewpoint-blind for every person). (2) `amni/serve/manipulation_flag.py` — technique-based honeytrap detector (moral-shaming / thought-terminating / manufactured-consensus / flattery / othering), direction-agnostic so it cannot be dodged by switching costumes; honeytraps from any side score ~85, honest argument ~20. - v6.11.50 — Physical-harm gate (`amni/serve/harm_semantic.py`) — Anthony's "Ten Commandments + one rule" design. On top of the baked commandments/laws, ONE added rule: block content only when a focused model judgment says it is >60% likely to CAUSE or ENABLE real PHYSICAL harm/death to a person (self or others). PHYSICAL ONLY — emotional/mental is explicitly out of scope. Low false positives by design: a cheap keyword/semantic prefilter decides only whether to ASK; the judgment is DIRECTION-aware (prevent/defend/treat/educate/discuss = ~0, only enabling harm scores high). Validated on an adversarial gauntlet: 0 false positives (self-defense, suicide-prevention, poison-ivy treatment, mining explosives, atomic-bomb history all pass; benign max conf 30), 7/7 genuine harm blocked (min conf 70) — clean 30<->70 separation. Replaces the brittle pure-keyword approach as the harm signal; complements, never replaces, the AsimovLayer. (2026-06-04)
