# Loop: security hardening — PII leak prevention + malicious code-run prevention

Runs alongside the coding-knowledge loop. Each wake: read this, harden one concrete thing, test it, update, schedule next. Builds stay server-down.

## Threat model
- **Malicious code run**: Adam runs generated AND crawled code in the debug/test cycle (`self_debug.run_in_sandbox`, `run_python` skill). A poisoned GitHub/web source or adversarial prompt could execute file-deletion / shell / network / sandbox-escape code on the host.
- **PII leak**: personal data reaching PTEX/federated files. Boundary validated (record-time local-slot routing + is_publishable + scrub_pii + drop-on-flag), but leans on `detect_personal` regex coverage; manual `/teach` ungated; crawled PII scrubbed only at publish.
